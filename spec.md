# Tool App Design Document

## 一、产品概览

| 维度 | 值 |
|------|-----|
| 产品名称 | 便签整理术 |
| 技能来源 | 《人人都能做到的便签整理术》（须藤由美） |
| 技能总数 | 10 个 procedural skills + 25 个 factual SKUs |
| 产品类型 | 效率工具（时间管理 + 整理收纳 + 习惯养成） |
| 交互方式 | 组合操控（鼠标点击 + 拖拽操作） |
| 核心主题 | 用便签方法论将「没时间整理」变成「随时能动手」 |
| 目标体验 | 用户 3 分钟内上手核心功能 |

---

## 二、目标人群

### 核心用户

| 维度 | 描述 |
|------|------|
| 画像 | 25-45 岁家庭主妇、自由职业者，非技术背景，希望改善家庭整理和时间管理 |
| 使用频率 | 每日（查看时间轴、打卡）+ 按需（创建便签、拆解任务） |
| 设备偏好 | 桌面浏览器为主，手机浏览器为辅 |
| 技术水平 | 无技术背景，需要直观的操作引导 |

### 次要用户

家庭成员（丈夫、孩子），通过「家庭任务」功能被邀请参与整理，偶尔使用。

### 使用场景

| 场景 | 时机 | 目的 |
|------|------|------|
| 晨间规划 | 每天早上打开工具 | 查看时间轴，将 TODO 便签安排到空闲时段 |
| 整理启动 | 决定开始整理某个区域时 | 用妖怪拆解把大任务拆成小步骤，用方阵确定优先级 |
| 日常打卡 | 每天睡前 | 在 21 天打卡中记录当日整理习惯完成情况 |
| 购物补货 | 发现日用品备用被启用时 | 在库存管理中标记，自动加入购物清单 |

## 三、产品目的

### 用户痛点

用户知道家里需要整理，但面对杂乱无章的环境感到「不知从何下手」「没时间做」「做了又会乱回去」。传统的纸质便签和备忘录容易丢失、无法分类检索、无法追踪进度，且家庭成员难以协作参与。

### 核心价值

这个工具帮用户将书中的便签整理方法论数字化（可视化时间、细分大任务、分类待办、追踪习惯），从而把模糊的「该整理了」变成具体的、可立即执行的行动清单。

### 成功标准

- 用户能把一个模糊的大任务拆解为 3 个以上可执行的小步骤
- 用户能在时间轴上识别出至少 2 个可利用的零碎时间段
- 用户能连续 7 天以上使用打卡功能
- 库存管理能自动提醒用户补货，不再出现「突然发现没了」的情况

## 四、视觉风格规范

### 设计风格选择

**极简克制**：以最少量的视觉元素构建画面，严格遵循"少即是多"的原则；去除所有非必要的装饰、纹理、渐变和复杂结构；采用大量留白、精简的色彩组合、清晰的几何形态和精准的排版布局；强调元素间的空间关系与比例平衡，通过克制的设计语言传递清晰的信息层级和宁静的视觉体验。

### 配色方案

| 用途 | 颜色值 | 说明 |
|------|--------|------|
| 60% 主色（背景/表面） | #FAFAF7 | 暖白，营造纸面般的温暖基调 |
| 30% 辅助色（文字/边框） | #1A1A1A | 碳黑，建立清晰的文字层级 |
| 10% 强调色（按钮/高亮） | #C67A4B | 赤陶，温暖而不刺眼的行动引导 |
| 成功色 | #3A7D1A | 森绿，用于完成状态和备用充足 |
| 警告/危险色 | #D44444 | 朱红，用于删除和需补货状态 |
| 辅助文字色 | #6B6B6B | 中灰，用于次要信息和提示 |

> 便签卡片保留 6 色系：暖黄 #FFF9E0、柔粉 #FFF0F2、浅绿 #EEFAE8、天蓝 #EBF5FF、淡橘 #FFF4E8、淡紫 #F5F0FA。

### 排版规范

| 用途 | 字体 | 字号 | 字重 |
|------|------|------|------|
| 标题 | PingFang SC / Noto Sans SC | 17-20px | 600 |
| 正文 | PingFang SC / Noto Sans SC | 14-16px | 400 |
| 辅助文字 | 同正文 | 12-13px | 400/500 |

### 间距与圆角

- **间距基准**：8px 网格（8 / 16 / 24 / 32 / 40）
- **卡片内边距**：24px
- **模块间距**：16px
- **圆角**：4-8px（中性，不过于硬朗也不过于圆润）

### 必须实现的视觉反馈

| 用户操作 | 视觉反馈要求 |
|---------|------------|
| 创建便签 | 新便签淡入 + 微位移入场 |
| 删除便签 | 淡出消失 |
| 打卡完成 | 圆点从边框态变为填充态（赤陶色），数字计数更新 |
| 拖拽便签到时间轴/方阵 | 便签半透明跟随，目标区域高亮提示 |
| 悬停便签 | 微抬升（translateY -2px）+ 阴影加深 |
| 视图切换 | 整体淡入过渡 250ms |
| 空状态 | 居中引导文案，告诉用户下一步操作 |

### 动效标准

- 所有过渡时长 220ms，缓动曲线 cubic-bezier(0.25, 0.1, 0.25, 1)
- 尊重 `prefers-reduced-motion` 系统偏好
- 动效仅用于状态反馈和视图过渡，不做装饰性动画

## 五、功能清单

> **重要：完整的技能数据文件位于项目根目录的 `procedural/` 文件夹中（skill_001 至 skill_010 的 SKILL.md）。构建应用时，必须读取每个 skill 文件的完整内容，将其方法论融入功能逻辑，而非仅依赖以下摘要。**

| 优先级 | 功能名称 | 对应技能 | 解决什么问题 | 输入 | 输出 |
|--------|----------|---------|-------------|------|------|
| P0 | 便签板 | skill_001 便签整理法 | 脑中想法无处记录、容易遗忘 | 文字 + 颜色 + 类型（TODO/日程） | 可视化的彩色便签卡片 |
| P0 | 时间轴 | skill_002 时间可视化 | 不知道哪里有空闲时间 | 日程块标记 + 拖入便签 | 一天的时间分配全景图 |
| P0 | 待办清单 | skill_003 待完成清单 | 琐事多但不知什么时候做 | 便签按场景分类 | 按情境可查的任务列表 |
| P0 | 妖怪拆解 | skill_004 任务细分 | 大任务让人拖延不敢动手 | 一个模糊的大任务 | 可立即执行的小步骤树 |
| P1 | 整理方阵 | skill_005 + skill_006 + skill_007 | 不知道先整理哪里、怎么安排 | 拖拽便签到四象限 | 优先级排序 + 时间精力匹配 |
| P1 | 21天打卡 | skill_009 习惯养成 | 整理做了几天就放弃 | 设定习惯目标 + 每日打卡 | 21天进度可视化 + 达成奖励 |
| P1 | 库存管理 | skill_008 循环便签 | 日用品突然用完才发现没备用 | 物品 + 备用状态 | 自动购物清单 |
| P2 | 家庭任务 | skill_010 家庭动员 | 整理只有一个人干，家人不参与 | 任务分配到成员 | 完成计分板 |

## 六、用户动线

### 核心路径

```
打开 → 便签板创建便签 → 拖入时间轴安排时间 → 大任务用妖怪拆解细分 → 用方阵排优先级 → 每日打卡追踪习惯 → 库存管理自动提醒补货
```

### 功能流转关系

| 源功能 | 流向 | 目标功能 | 流转方式 |
|--------|------|---------|---------|
| 便签板 | → | 时间轴 | 拖拽（TODO 便签拖入时间段） |
| 便签板 | → | 整理方阵 | 拖拽（便签拖入四象限） |
| 便签板 | → | 待办清单 | 自动（按便签的分类字段归入对应类别） |
| 库存管理 | → | 购物清单 | 自动（启用备用时，物品自动进入购物清单） |
| 购物清单 | → | 库存管理 | 点击（标记已购买后，物品恢复有备用状态） |

## 七、信息结构

### 导航结构

```
├── 便签板        （skill_001 — 核心入口，创建和管理所有便签）
├── 时间轴        （skill_002 — 每日时间可视化与安排）
├── 待办清单      （skill_003 — 按场景分类的待办列表）
├── 妖怪拆解      （skill_004 — 大任务拆解为小步骤）
├── 整理方阵      （skill_005 + 006 + 007 — 优先方阵 / 时间方阵 双 tab）
├── 21天打卡      （skill_009 — 习惯养成追踪）
├── 库存管理      （skill_008 — 循环便签与购物清单）
└── 家庭任务      （skill_010 — 家庭成员任务分配）
```

### 合并与拆分决策

| 功能 | 决策 | 原因 |
|------|------|------|
| skill_005 四步整理 + skill_006 适量化 + skill_007 定位贴标 | 合并为「整理方阵」 | 三者是整理流程的连续步骤，通过优先方阵和时间方阵统一呈现排序逻辑 |
| skill_001 便签整理法 | 独立为「便签板」 | 是所有功能的数据源头，需要独立的创建/管理入口 |
| skill_004 任务细分 | 独立为「妖怪拆解」 | 使用时机与日常便签管理不同，是专门面对大任务时启动的工具 |
| skill_008 库存管理 | 独立 | 管理对象是物品库存而非任务，数据模型完全不同 |

## 八、数据设计

### 存储方案

| 维度 | 选择 |
|------|------|
| 存储方式 | localStorage |
| 多端同步 | 否（单机使用，家庭成员共用同一设备） |
| 数据上限 | 5MB（localStorage 标准上限，约可存储数千条便签） |
| 导入导出 | 不支持（v1.0 暂不实现） |

### 核心数据模型

```json
{
  "notes": [{
    "id": "string",
    "text": "string",
    "color": "yellow | pink | green | blue | orange | purple",
    "type": "todo | schedule",
    "category": "string | empty",
    "createdAt": "timestamp"
  }],
  "timelineBlocks": [{
    "id": "string",
    "text": "string",
    "hour": "number (6-23)",
    "date": "YYYY-MM-DD"
  }],
  "timelineNotes": [{
    "id": "string",
    "noteId": "notes.id",
    "hour": "number",
    "date": "YYYY-MM-DD"
  }],
  "monsters": [{
    "id": "string",
    "text": "string",
    "children": ["string"]
  }],
  "matrixPriority": { "A": ["noteId"], "B": ["noteId"], "C": ["noteId"], "D": ["noteId"] },
  "matrixTime": { "A": ["noteId"], "B": ["noteId"], "C": ["noteId"], "D": ["noteId"] },
  "habits": [{
    "id": "string",
    "goal": "string",
    "reward": "string",
    "startDate": "timestamp",
    "days": ["boolean x 21"]
  }],
  "inventory": [{
    "id": "string",
    "name": "string",
    "hasBackup": "boolean",
    "inShoppingList": "boolean"
  }],
  "familyTasks": [{
    "id": "string",
    "text": "string",
    "assignee": "dad | mom | child",
    "done": "boolean"
  }],
  "categories": ["string"]
}
```

### 数据关系

| 实体 A | 关系 | 实体 B | 说明 |
|--------|------|--------|------|
| notes | 一对多 | timelineNotes | 一条便签可被安排到多个时间段 |
| notes | 多对多 | matrixPriority / matrixTime | 便签可被分配到方阵象限 |
| notes | 一对多 | categories | 便签通过 category 字段归入待办分类 |
| inventory | 一对一 | 购物清单 | 启用备用时自动流转，购买后流转回 |

## 九、交互设计

### 交互模式选择

**本项目选择**：混合（新手友好为主，拖拽为辅）

**选择原因**：核心用户为非技术背景人群，创建/编辑操作使用弹窗表单降低学习成本；归类到时间轴和方阵使用拖拽以提升效率和直觉性。

### 操作映射

| 操作 | 实现方案 | 说明 |
|------|---------|------|
| 创建便签 | 弹窗表单 | 点击「+ 新建便签」按钮，填写内容/颜色/类型/分类 |
| 编辑便签 | 弹窗表单 | 悬停便签出现编辑按钮，点击打开预填表单 |
| 删除便签 | 直接删除 | 悬停出现删除按钮，点击即删，同步清理关联数据 |
| 归类到时间轴 | 拖拽 | 从待安排池拖拽便签到对应时间行 |
| 归类到方阵 | 拖拽 | 从底部便签池拖拽到四象限格子 |
| 妖怪拆解添加步骤 | 行内输入 | 输入框 + 回车即可添加，降低操作步骤 |
| 家庭任务添加 | 行内输入 | 每个成员列底部有输入框 + 回车添加 |
| 打卡 | 直接点击 | 点击日期圆点即切换完成状态 |

### 交互反馈

| 操作 | 视觉反馈 | 状态变化 |
|------|---------|---------|
| 创建便签 | 新卡片淡入出现 | notes 数组新增一条 |
| 拖拽便签 | 原位置半透明，目标区域高亮 | timelineNotes 或 matrix 新增关联 |
| 打卡点击 | 圆点从空心变为赤陶色填充 | habits.days[i] 翻转布尔值 |
| 启用备用 | 状态标签从「有备用」变为「需补货」 | hasBackup=false, inShoppingList=true |
| 完成家庭任务 | 文字划线 + 透明度降低 | familyTasks.done=true |

## 十、技能映射方案

### 映射逻辑

每个 procedural skill 直接映射为工具的一个独立功能视图，skill 中的步骤转化为功能的操作流程。factual SKU 中的知识点（如妖怪便签特征、时间段方阵定义、适量化窍门等）融入对应功能的 UI 提示文案和操作引导中。

### 映射详情

| 技能 | 描述 | 对应功能 | 功能中的体现 |
|------|------|---------|------------|
| skill_001 sticky-note-organization-method | 三大技巧：可视化、细分化、一元化 | 便签板 | 便签创建/管理的核心入口，所有信息集中在一处（一元化） |
| skill_002 time-visualization-scheduling | 时间轴标记日程，识别零碎时间 | 时间轴 | 6:00-23:00 时间线，日程块标记 + TODO 拖入空闲段（可视化） |
| skill_003 create-pending-task-list | 按时间段/场所分类的待完成清单 | 待办清单 | 4 个默认分类 + 自定义分类，便签自动归入 |
| skill_004 task-subdivision-technique | 识别妖怪便签，拆解为可执行小步骤 | 妖怪拆解 | 左侧大任务列表 + 右侧树形步骤拆解 + 引导提示（细分化） |
| skill_005 four-step-decluttering-process | 定目标→定顺序→适量化→定位置 | 整理方阵 | 优先方阵按压力×影响排序，指导用户先做什么（定顺序） |
| skill_006 item-quantity-optimization | 从零开始思维，确定物品适量 | 整理方阵 | 方阵辅助用户判断任务量级（适量化思维融入排序逻辑） |
| skill_007 assigning-homes-and-labeling | 给物品定位置，贴标签可视化 | 整理方阵 | 时间方阵按时间×人力分类，帮用户匹配当前可用资源（定位置） |
| skill_008 inventory-management-cyclic-notes | 循环便签管理库存，自动触发补货 | 库存管理 | 备用品启用→自动加入购物清单→购买后循环回备用状态 |
| skill_009 habit-formation-21-day-rule | 21 天打卡 + 奖励机制养成习惯 | 21天打卡 | 设定目标 + 奖励，21 个圆点逐日打卡，进度条 + 达成庆祝 |
| skill_010 family-decluttering-mobilization | 家庭会议 + 任务分配 + 游戏化 | 家庭任务 | 爸爸/妈妈/孩子三列任务板 + 完成计分 + 行内快速添加 |

## 十一、技术约束

| 维度 | 选择 |
|------|------|
| 技术栈 | 纯前端单 HTML 文件（内嵌 CSS + JS） |
